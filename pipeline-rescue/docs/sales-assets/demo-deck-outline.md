# Demo Deck Outline

## Slide 1: The Problem

Title: Deals do not usually die all at once.

Bullets:

- They go quiet.
- Next steps disappear.
- Managers notice too late.
- Reps do not always know what to rescue first.

Speaker note:

Most teams already have the data in HubSpot. The problem is turning that data into daily action.

## Slide 2: The Wedge

Title: Pipeline Rescue focuses only on stalled deal recovery.

Bullets:

- not a generic AI sales assistant
- not autonomous outbound
- not forecasting
- focused on deals already in pipeline

Speaker note:

The narrow focus makes the value easier to test, explain, and measure.

## Slide 3: The Workflow

Title: From stale deal to next action.

Flow:

1. Find at-risk deals.
2. Explain why each deal is at risk.
3. Recommend the next action.
4. Create the HubSpot task.
5. Draft the follow-up.
6. Capture operator feedback.

Speaker note:

Everything stays human-reviewed. The tool does not send messages automatically.

## Slide 4: The Pilot

Title: 30-day controlled pilot.

Bullets:

- HubSpot connection
- live deal queue
- task and note write-back
- weekly review
- measured adoption
- clear go/no-go criteria

Pricing:

- EUR500 setup
- EUR299 for 30 days
- EUR299/month after pilot

## Slide 5: Decision

Title: Start with one live pipeline.

Ask:

- connect HubSpot
- analyze real stale deals
- review output weekly
- decide after 30 days

Close:

If the recommendations are not credible, we stop. If the workflow creates useful rescue activity, we continue.
